# Modelos

Esta pasta contém os modelos de dados da aplicação. Cada modelo representa uma entidade do sistema (ex: Usuário, Grupo, Post) e é responsável por:

1.  **Definir a Estrutura:** Especifica os campos e tipos de dados de uma entidade.
2.  **Encapsular Regras de Negócio:** Centraliza a lógica de validação e de negócio (ex: um grupo pago deve ter preço maior que zero).
3.  **Prover uma Interface Consistente:** Garante que todas as partes do sistema acessem e manipulem os dados da mesma forma.

## Fluxo de Dados

Controle → Serviço → **Modelo** → Repositório → Consulta

O modelo atua como uma ponte entre os serviços e os repositórios, garantindo que os dados estejam sempre em um estado válido e consistente.
