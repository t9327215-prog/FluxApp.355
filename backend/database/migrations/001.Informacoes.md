📜 Diretrizes Obrigatórias 📜
─────────────────────────────
A camada de Migrations contém os scripts SQL que definem a estrutura (schema) do banco de dados.
Os arquivos são scripts SQL puros, versionados e numerados para garantir a ordem de execução.
A nomenclatura padrão é `NN_Descricao.da.Tabela.sql`.
Uma vez aplicada, uma migração não deve ser alterada; novas alterações exigem um novo arquivo de migração.
As migrações são executadas automaticamente na inicialização do backend para manter o banco de dados sincronizado.

🔄 Fluxo de Dados 🔄
─────────────────────────────
O fluxo começa na inicialização do servidor backend.
O sistema lê todos os arquivos de migração em ordem numérica.
O sistema executa sequencialmente cada migração que ainda não foi aplicada ao banco de dados.
O fluxo garante que a estrutura do banco de dados corresponda à versão mais recente definida no código.

⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de Definição de Schema: Criar, alterar ou remover tabelas e outras estruturas do banco (`CREATE TABLE`, `ALTER TABLE`).
Responsabilidade de Estrutura de Dados: Definir colunas, tipos de dados, restrições (`NOT NULL`) e chaves (`PRIMARY KEY`).
Responsabilidade de Relacionamentos: Estabelecer chaves estrangeiras (`FOREIGN KEY`) para garantir a integridade referencial.
Responsabilidade de Versionamento do Banco de Dados: Agir como um controle de versão para a estrutura do banco, permitindo evoluções e rastreabilidade.
