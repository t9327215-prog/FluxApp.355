📜 Diretrizes Obrigatórias 📜
─────────────────────────────
A camada de Query armazena exclusivamente as strings de consulta SQL.
A camada de Query não executa nenhuma consulta no banco de dados.
A camada de Query atua como uma biblioteca de consultas para a aplicação.
A camada de Query é importada pela camada de Repositórios.
A camada de Query centraliza todo o código SQL, facilitando a manutenção.
A camada de Query garante a separação entre a lógica de banco de dados e a de aplicação.

🔄 Fluxo de Dados 🔄
─────────────────────────────
O fluxo não se aplica diretamente; esta camada é um repositório estático.
As strings de query são exportadas como constantes.
A camada de Repositórios importa essas constantes para usá-las na execução de queries.

⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de armazenamento de queries define e exporta todas as consultas SQL.
Responsabilidade de centralização de SQL mantém todo o código SQL em um único local.
Responsabilidade de nomeação de queries fornece nomes claros e consistentes para cada consulta.
Responsabilidade de desacoplamento isola o SQL do código JavaScript que o executa.
