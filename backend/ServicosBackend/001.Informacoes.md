📜 Diretrizes Obrigatórias 📜
─────────────────────────────
A camada de Serviços Backend é o cérebro da aplicação.
A camada de Serviços Backend contém a lógica de negócio principal.
A camada de Serviços Backend orquestra as operações e toma as decisões.
A camada de Serviços Backend é chamada pela camada de Controles.
A camada de Serviços Backend importa a camada de Repositórios.
A camada de Serviços Backend é responsável por gerenciar transações.

🔄 Fluxo de Dados 🔄
─────────────────────────────
Fluxo de dados começa quando um Controle chama um Serviço Backend.
Fluxo de dados continua com a orquestração de chamadas para um ou mais Repositórios.
Fluxo de dados aplica regras de negócio, cálculos e manipulações de dados.
Fluxo de dados termina quando o Serviço retorna o resultado final para o Controle.

⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de orquestração de operações chama múltiplos repositórios em sequência.
Responsabilidade de implementação de regras de negócio lida com validações complexas e cálculos.
Responsabilidade de transações garante que operações complexas sejam atômicas (tudo ou nada).
Responsabilidade de separação de responsabilidades mantém os Controles "magros" e os Repositórios "burros".
