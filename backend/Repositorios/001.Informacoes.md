📜 Diretrizes Obrigatórias 📜
─────────────────────────────
A camada de Repositórios representa o nível mais baixo de acesso a dados no backend.
A camada de Repositórios tem função exclusiva de executar queries SQL puras.
A camada de Repositórios interage diretamente com o banco de dados.
A camada de Repositórios é importada pela camada de Serviços Backend.
A camada de Repositórios importa os arquivos de Query.
A camada de Repositórios não deve conter nenhuma lógica de negócio.
A camada de Repositórios não deve conter validação de permissões.
A camada de Repositórios não deve conter orquestração de múltiplas operações.
Os dados recebidos são considerados prontos para construir queries (INSERT, SELECT, UPDATE, DELETE).
🔄 Fluxo de Dados 🔄
─────────────────────────────
Fluxo de dados começa na camada de Repositórios chamada pelo Serviço Backend.
Fluxo de dados continua com a execução de uma query SQL atômica no banco de dados.
Fluxo de dados termina retornando o resultado bruto da query para o Serviço que chamou.
⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de execução de Queries constrói e executa comandos SQL (INSERT, SELECT, UPDATE, DELETE).
Responsabilidade de execução de Queries realiza operações atômicas no banco de dados.
Responsabilidade de abstração do Banco de Dados isola o resto da aplicação dos detalhes do banco de dados.
Responsabilidade de abstração do Banco de Dados garante que apenas esta camada fale diretamente com o banco.
Responsabilidade de Nenhuma Lógica de Negócio evita validações, orquestração ou regras de negócio na camada.
