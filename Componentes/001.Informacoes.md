📜 Diretrizes Obrigatórias 📜
─────────────────────────────
A pasta `Componentes` contém todos os componentes React reutilizáveis, organizados semanticamente por funcionalidade (ex: `ComponentesDeFeed`, `ComponentesDeGroups`).
A maioria dos componentes deve ser "burra" (dumb), ou seja, focada apenas na apresentação. Eles recebem dados e funções de callback via `props` e não devem buscar seus próprios dados.
Componentes de UI genéricos e agnósticos ao domínio (ex: `Button`, `Input`, `Modal`) devem residir na subpasta `ComponenteDeInterfaceDeUsuario`.
Componentes não devem ter lógica de negócio; essa responsabilidade é dos `Hooks` e `Pages`.

🔄 Fluxo de Dados 🔄
─────────────────────────────
Uma `Page` consome um `Hook` para obter dados e lógica.
A `Page` renderiza componentes de domínio (ex: `<Container.Feed>`), passando os dados e funções do Hook via `props`.
O componente de domínio pode renderizar componentes de UI menores, repassando as `props`.
Quando um usuário interage com um componente (ex: um clique de botão), a função de callback recebida (`onClick`) é acionada.
Essa chamada "sobe" a árvore de componentes até o `Hook` ou `Page` que detém a lógica, que então executa a ação necessária (ex: chamada de API).

⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de Apresentação Visual: Renderizar a UI com base nas `props` recebidas, aplicando os estilos necessários.
Responsabilidade de Reutilização: Ser genérico o suficiente para ser usado em diferentes partes da aplicação sem modificação.
Responsabilidade de Isolar a Lógica de Apresentação: Manter uma separação clara entre a aparência (o que o componente faz) e a lógica de negócio (como os dados são obtidos ou manipulados).
