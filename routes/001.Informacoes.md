📜 Diretrizes Obrigatórias 📜
─────────────────────────────
A pasta `routes` mapeia as URLs do navegador para os componentes de `pages` correspondentes, usando `react-router-dom`.
As rotas são modularizadas por domínio (ex: `Rotas.Feed.tsx`, `Rotas.Grupos.tsx`) para organização e escalabilidade.
O componente `ProtectedRoute.tsx` deve ser usado para envolver rotas que exigem autenticação, redirecionando usuários não logados para a página de login.
Parâmetros dinâmicos (ex: `/perfil/:userId`) devem ser usados para passar dados através da URL para as páginas.

🔄 Fluxo de Dados 🔄
─────────────────────────────
Quando a aplicação carrega, o `react-router-dom` lê a URL atual do navegador.
O roteador busca uma correspondência nas definições de rotas.
Se for uma `ProtectedRoute`, ele verifica o status de autenticação do usuário (via `Hook` ou `Context`). Se o usuário não estiver autenticado, é redirecionado para `/login`.
Se o usuário estiver autenticado (ou a rota for pública), o componente de `Page` correspondente é renderizado.
Navegações subsequentes via componentes `Link` repetem o processo sem recarregar a página, garantindo a experiência de SPA.

⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de Mapeamento URL -> Componente: Definir qual `Page` renderizar para cada URL da aplicação.
Responsabilidade de Controle de Acesso: Proteger rotas sensíveis, garantindo que apenas usuários autenticados possam acessá-las.
Responsabilidade de Extração de Parâmetros de Rota: Disponibilizar parâmetros dinâmicos da URL para os componentes de `Page`.
Responsabilidade de Navegação: Facilitar tanto a navegação declarativa (via `Link`) quanto a programática (via `useNavigate`).
