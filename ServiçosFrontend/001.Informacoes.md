📜 Diretrizes Obrigatórias 📜
─────────────────────────────
A pasta `ServiçosFrontend` isola toda a comunicação com APIs externas, como o backend, e outros serviços de terceiros.
Cada serviço deve encapsular a lógica de uma entidade ou funcionalidade específica (ex: `Sistema.Autenticacao.Supremo`, `groupService`).
Os serviços são agnósticos em relação à UI. Eles não devem importar `hooks` ou `componentes` do React.
Os métodos dos serviços devem retornar promessas, lidando com a lógica de `fetch`, `axios` ou qualquer cliente HTTP.
Devem tratar a formatação dos dados enviados e recebidos da API, garantindo a consistência com os `tipos` do projeto.

🔄 Fluxo de Dados 🔄
─────────────────────────────
Um `Hook` precisa buscar ou enviar dados (ex: `useFeed` para buscar posts).
O `Hook` chama um método do serviço correspondente (ex: `feedService.getPosts()`).
O método do serviço monta a requisição (URL, headers, body) e a envia para o endpoint da API.
O serviço aguarda a resposta da API.
Ao receber a resposta, o serviço pode realizar um pré-processamento (ex: renomear campos) e então retorna os dados (ou um erro) para o `Hook` que o chamou.

⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de Comunicação com API: Centralizar toda a lógica de `fetch` ou `axios` para interagir com o backend.
Responsabilidade de Abstração da Fonte de Dados: Os `Hooks` não sabem *como* os dados são obtidos; eles apenas pedem os dados ao serviço. Isso facilita a troca da fonte de dados (ex: de REST para GraphQL) no futuro.
Responsabilidade de Tratamento de Erros de Rede: Capturar erros de rede (ex: 404, 500) e padronizar a forma como são retornados para a camada de `Hooks`.
Responsabilidade de Transformação de Dados: Adaptar os dados recebidos da API para o formato esperado pela aplicação (definido nos `tipos`) e vice-versa.
