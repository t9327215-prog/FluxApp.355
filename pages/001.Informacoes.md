📜 Diretrizes Obrigatórias 📜
─────────────────────────────
Os arquivos na pasta `pages` representam componentes de tela de mais alto nível (uma página completa).
Cada página atua como um "maestro", orquestrando `Hooks` e `Componentes` para construir a UI.
As páginas são as principais consumidoras dos `Custom Hooks` para buscar dados e gerenciar estado.
Definem a estrutura de alto nível e utilizam componentes de `layout` para consistência visual.
São o ponto de entrada para as rotas definidas na pasta `routes`.

🔄 Fluxo de Dados 🔄
─────────────────────────────
O fluxo começa quando um usuário navega para uma URL (ex: `/feed`).
O sistema de `routes` renderiza o componente de página associado (ex: `Feed.tsx`).
A página chama os `Hooks` necessários (ex: `useFeed()`) para buscar dados.
O Hook busca os dados (via `ServiçosFrontend`) e retorna o estado (dados, carregando, erro).
A página recebe os dados e os passa via `props` para os `Componentes` de apresentação.
A página é renderizada, exibindo a UI completa e reativa para o usuário.

⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de Composição da Tela: Montar a tela completa, organizando os componentes necessários.
Responsabilidade de Conexão com a Lógica: Consumir `Hooks` para obter estado, dados e funções de manipulação.
Responsabilidade de Estrutura e Layout: Definir o layout geral da página, garantindo a consistência da interface.
Responsabilidade de Passagem de Dados (Props Drilling): Distribuir os dados e funções dos Hooks para os componentes filhos.
