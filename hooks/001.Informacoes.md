📜 Diretrizes Obrigatórias 📜
─────────────────────────────
A pasta `hooks` contém Custom Hooks do React, nomeados com o prefixo `use`.
Hooks são o principal mecanismo para encapsular, reutilizar e gerenciar a lógica de estado.
Hooks são a fonte da verdade para o estado da aplicação e gerenciam a busca de dados (data fetching).
Toda lógica que interage com o "mundo exterior" (APIs, timers, DOM) deve ser gerenciada em `useEffect` dentro de um Hook.
A reutilização de lógica entre componentes deve ser feita criando um Custom Hook, evitando duplicação.

🔄 Fluxo de Dados 🔄
─────────────────────────────
Um componente de UI (em `pages` ou `components`) chama um Hook para obter dados e funções.
Dentro do Hook, um `useEffect` chama um serviço da camada `ServiçosFrontend` para buscar ou enviar dados.
O serviço executa a chamada de API, e o Hook gerencia os estados da requisição (carregando, sucesso, erro).
O Hook atualiza seu estado interno com os dados recebidos, o que faz o componente de UI ser renderizado novamente.
O componente renderiza a informação atualizada, conectando a camada de serviços à de apresentação.

⚡ Responsabilidades ⚡
─────────────────────────────
Responsabilidade de Gerenciamento de Estado: Usar `useState` e `useReducer` para controlar o estado local da funcionalidade.
Responsabilidade de Busca de Dados: Orquestrar chamadas à API, chamando os serviços correspondentes e tratando o ciclo de vida da requisição.
Responsabilidade de Efeitos Colaterais (Side Effects): Isolar interações com sistemas externos (APIs, armazenamento local) do resto da lógica do componente.
Responsabilidade de Reutilização de Lógica: Encapsular lógicas complexas que são necessárias em múltiplos componentes, provendo uma API simples para consumo.
